Comment saisir un timesheet
Procédure pas-à-pas dans WinBIZ Cloud. À suivre pour chaque entrée, chaque jour, 15 minutes avant la fin de la journée.
⏱ Règle absolue — saisie quotidienne
  • Saisir ses heures 15 minutes avant la fin de chaque journée — pas en fin de semaine
  • Une description saisie le jour même est précise et facturable
  • Saisie de mémoire 3 jours après = description vague = heure non facturable
  • Deadline de soumission : vendredi 12h00 — c'est la limite de validation, pas la date de saisie
Étapes de saisie dans WinBIZ Cloud
  1. 1
    Ouvrir WinBIZ Cloud → Module Temps & Prestations
    Naviguer vers le module de saisie des heures. Si le module n'est pas visible, contacter Katja.
  2. 2
    Cliquer sur "Nouvelle entrée" ou "+"
    Une fiche de saisie s'ouvre. Ne pas modifier une entrée existante d'un autre collaborateur.
  3. 3
    Sélectionner la date
    Toujours la date du jour où le travail a été effectué. Pas la date de saisie si différente.
    Ne jamais grouper plusieurs jours en une seule entrée
  4. 4
    Sélectionner le client / numéro de dossier
    Rechercher par nom ou numéro de dossier. Vérifier que c'est le bon dossier avant de continuer.
    Champ obligatoire — sans client, l'entrée est rejetée
  5. 5
    Sélectionner le code type de prestation
    Choisir dans la liste déroulante le code correspondant au travail effectué. Voir onglet "Codes & types" pour la liste complète.
    Voir liste des codes dans l'onglet 2
  6. 6
    Saisir la durée — arrondie au quart d'heure
    Format : HH:MM. Arrondir au quart d'heure le plus proche (0:15 / 0:30 / 0:45 / 1:00).
    Durée > 4h sur une seule entrée → ajouter une justification dans la description
  7. 7
    Rédiger la description — champ le plus important
    Décrire précisément le travail effectué. Minimum 8 mots. Katja doit pouvoir comprendre le travail sans vous recontacter.
    Voir exemples dans l'onglet 5
  8. 8
    Enregistrer l'entrée
    Cliquer sur "Enregistrer" ou "Valider". L'entrée apparaît dans la liste des timesheets de la semaine.
  9. 9
    Soumettre avant vendredi 12h00
    Les entrées non soumises à temps ne sont pas traitées dans le cycle de facturation de la semaine.
    Deadline non respectée = retard de facturation client
Les 4 champs obligatoires — résumé
Champs obligatoires
  • 1
    Date — jour du travail effectué
  • 2
    Client / Dossier — numéro WinBIZ correct
  • 3
    Code prestation — type de travail effectué
  • 4
    Description — travail précis, min. 8 mots
⚠ Ce qui rend une entrée non facturable
  • Description manquante ou générique (< 5 mots)
  • Client / dossier manquant ou incorrect
  • Code prestation absent
  • Durée > 4h sans justification
  • Non soumise avant vendredi 12h00
  • Non clarifiée dans les 24h après demande Katja

Règles d'utilisation des codes
En cas de doute sur le code
  • Utiliser le code le plus spécifique disponible
  • En cas d'hésitation entre COMPTA et FISCAL : choisir selon l'objet principal du travail
  • Un appel téléphonique sur un sujet fiscal → CONS si > 15 min, FISCAL si c'est de la saisie
  • Si vraiment incertain : utiliser INTERN et préciser dans la description — Katja reclassifiera
⚠ Code CONS — attention TVA
  • Le code CONS est le plus souvent mal configuré en TVA dans WinBIZ
  • Vérifier systématiquement que la TVA 8.1% est bien appliquée
  • Si TVA à 0% sur CONS → signaler immédiatement à Katja

Cas limites et situations ambiguës
Comment gérer les situations qui ne rentrent pas clairement dans les règles standard. En cas de doute persistant, contacter Katja.
Oublis et corrections
Cas fréquent J'ai oublié de saisir hier (ou avant-hier)
Saisir quand même avec la date du jour où le travail a été effectué, pas la date de saisie. Ajouter en début de description : "[Saisie différée — travail effectué le JJ/MM]". Katja acceptera la saisie mais signalera la récurrence si ça devient une habitude.
Cas fréquent J'ai fait une erreur sur une entrée déjà soumise
Ne pas modifier directement si Katja a déjà validé. Contacter Katja par message en précisant : numéro de dossier, date de l'entrée erronée, et la correction à apporter. Katja effectue la modification dans WinBIZ.
Cas fréquent J'ai travaillé sur deux dossiers différents en même temps
Créer deux entrées séparées avec la durée estimée pour chaque dossier. Estimer honnêtement la répartition. Ne jamais mettre deux dossiers dans une seule entrée.
Situations spéciales
Situation spéciale Travail en urgence (tarif × 2)
Utiliser le code URGENCE uniquement si le client a explicitement demandé un traitement en moins de 48h. Mentionner dans la description : "Urgence demandée par client le JJ/MM — traitement sous 48h". Ne pas utiliser URGENCE de sa propre initiative sans confirmation de Khaled ou Marcin.
Situation spéciale Réunion interne PHI (pas de client)
Code INTERN. Description : objet de la réunion, participants, durée. Ces heures ne sont pas facturées mais sont suivies pour la rentabilité interne. Ne pas les omettre.
Situation spéciale Appel téléphonique ou email client
Si > 15 minutes : saisir avec code CONS et décrire précisément l'objet de l'échange. Si < 15 minutes et isolé : peut être inclus dans une autre entrée du même dossier le même jour en ajoutant "+ appel client 10 min re: [sujet]" à la description.
Situation spéciale Travail hors forfait pour un client en forfait
Saisir normalement avec le code correspondant. Ajouter dans la description : "[HORS FORFAIT — à valider avec Katja]". Ne pas décider seul si c'est facturable ou non — c'est Katja + Meriem qui classifient à l'étape 3 du workflow.
Situation spéciale Formation ou autoformation professionnelle
Code INTERN. Description : sujet de la formation, source (cours, webinaire, lecture), lien avec l'activité Phi. Ces heures sont non facturables mais importantes à tracer pour le développement des compétences.
Situation spéciale Déplacement chez un client
Deux entrées séparées : (1) le déplacement avec code ADMIN et mention "Déplacement A/R client [ville] — CHF 160 forfait", (2) le temps de travail chez le client avec le code correspondant. Le forfait déplacement CHF 160 HT est fixe et distinct des heures de travail.

La règle des 3 éléments
Une bonne description contient toujours 3 éléments
  • Quoi — le travail effectué précisément (pas juste le type)
  • Sur quoi — les comptes, documents, périodes, nombre de pièces concernés
  • Résultat — ce qui a été produit, décidé, trouvé, ou envoyé
